
# Linux란 무엇인가

> “리눅스가 뭔지”, “왜 종류가 많은지”, “배포판은 뭐가 다른지” 정리

---

## 1) 리눅스(Linux)란?

리눅스는 **운영체제(OS) 커널(kernel)** 이다.

커널은 컴퓨터 자원을 관리하고 프로그램이 실행될 수 있게 해주는 “핵심 엔진”이며, 대표적으로 아래 일을 한다.

- **프로세스 관리**: 실행 중인 프로그램(프로세스)을 만들고 종료하고, CPU 시간을 배분한다.
- **메모리 관리**: RAM을 효율적으로 나눠 쓰게 하고, 부족하면 스왑 등으로 보완한다.
- **파일시스템/디스크 관리**: 파일 저장/읽기, 마운트, 권한 검사 등을 처리한다.
- **네트워크 처리**: TCP/IP 통신, 소켓, 라우팅 등 네트워크 기능의 핵심을 담당한다.
- **장치 제어**: 드라이버를 통해 디스크/네트워크 카드/USB 같은 하드웨어를 제어한다.
- **보안/권한**: 사용자/그룹/권한을 기반으로 접근을 통제한다.

즉, 리눅스는 “터미널 명령어 모음”이 아니라 **컴퓨터를 실제로 굴리는 OS의 핵심 부품(커널)** 이다.

---

## 2) 그런데 우리가 말하는 “리눅스 OS”는 뭘까?

현업에서 “리눅스”라고 말할 때는 보통 **리눅스 배포판(Distribution, Distro)** 을 의미한다.

### 배포판(Distro) = 커널 + 사용자 영역 도구 + 패키지 관리자 + 기본 설정 + 업데이트/지원 정책
- **Linux 커널**
- 기본 명령어 도구들(쉘, coreutils 등)과 시스템 구성요소
- **패키지 관리자**(소프트웨어 설치/업데이트 방식)
- 초기 설정, 서비스 구성 방식
- 릴리즈 주기, 안정성/최신성 방향, 보안 업데이트 정책, 지원 기간

그래서 배포판이 달라지면 “커널은 같아도” 운영 경험이 달라진다.

---

## 3) 리눅스 배포판 종류는?

배포판은 수백 개가 있지만, 실무에서 중요한 건 “계열”이다.  
계열은 주로 **패키지 시스템과 철학(안정/최신/지원 방식)** 으로 나뉜다.

### 대표 계열 (개념만)
- **Debian 계열**: 안정성과 범용성, 자료/사용자층이 매우 큼
- **Red Hat 계열**: 기업 환경에서 많이 채택, 운영 표준화/정책 중심
- **SUSE 계열**: 기업용 배포판 라인과 관리 도구 생태계
- **Rolling Release 계열**: 항상 최신(대신 운영 안정성/변경 관리 난이도 ↑)
- **경량/특수 목적**: 컨테이너 최적화, 보안 테스트 등 목적 특화

> 포인트: “어떤 배포판이 최고”가 아니라, **목적(운영/개발/보안/경량)** 에 맞춰 선택한다.

---

## 4) 왜 배포판이 이렇게 많을까?

배포판이 많아진 이유는 단순하다.  
**운영 철학과 우선순위가 다르기 때문**이다.

배포판이 달라질 때 보통 달라지는 것들:

1. **패키지 설치/업데이트 방식**
   - 어떤 저장소를 쓰고, 어떤 규칙으로 버전을 올리는지
2. **업데이트 정책**
   - 최신 기능을 빨리 받는 대신 변화가 잦은지
   - 안정성을 우선해서 변화가 적은지
3. **기본 보안/권한 설정**
   - 보안 기능의 기본 활성화 여부, 정책 템플릿 등
4. **서비스 관리/기본 구성**
   - 서비스(데몬) 구성 방식, 기본 설정 파일 위치/구조
5. **지원 정책**
   - 장기 지원(LTS) 여부, 기업/커뮤니티 지원 형태

---

## 5) 리눅스를 쓰는 이유 (서버에서 특히)

리눅스가 서버에서 표준처럼 쓰이는 이유는 “기능이 많아서”가 아니라  
**운영하기 유리한 구조** 덕분이다.

- **안정성**: 장시간 무중단 운영에 강함
- **자동화 친화**: 스크립트/CLI 중심으로 관리가 쉬움
- **생태계**: 웹서버, DB, 컨테이너, 쿠버네티스 등 서버 핵심 생태계가 리눅스 중심
- **비용 효율**: 라이선스/자원 관점에서 유연함
- **원격 운영 최적화**: SSH 기반으로 원격 관리가 자연스러움

---

## 6) 리눅스의 “기본 세계관” 6가지

리눅스를 이해할 때 아래 6개를 잡으면, 나머지는 따라온다.

### 1) 모든 것은 프로세스다
- 프로그램은 실행되면 **프로세스**가 된다.
- 서버 운영은 결국 “프로세스가 잘 떠있냐, 죽었냐, 먹통이냐”를 다루는 일이다.

### 2) 권한이 가장 먼저다
- 사용자(User) / 그룹(Group) / 권한(rwx)
- “안 된다”의 절반은 권한 문제다.

### 3) 로그가 답이다
- 장애/이슈는 로그에 흔적이 남는다.
- 어디 로그를 봐야 하는지 아는 게 실력이다.

### 4) 네트워크는 “경로 + 정책”이다
- 통신 문제는 보통 아래 중 하나다:
  - 서비스 미기동(포트 미리스닝)
  - 방화벽/보안 정책 차단
  - 라우팅/경로 문제
  - DNS 문제

### 5) 패키지 관리자 = 설치의 표준
- 리눅스는 설치가 “exe 다운로드”가 아니라
- 저장소 기반 설치/업데이트가 기본 흐름이다.

### 6) 설정은 텍스트로 관리한다
- 설정 파일은 대부분 텍스트이고,
- 버전관리(Git), 자동화(스크립트), 재현이 쉽다.

---

## 7) 용어

- **Kernel(커널)**: 운영체제 핵심 엔진
- **Distro(배포판)**: 커널 + 도구 + 패키지 관리자 + 정책의 완제품 OS
- **Shell(쉘)**: 명령어를 입력하는 환경(예: bash 등)
- **Daemon(데몬)**: 백그라운드에서 돌아가는 서비스 프로세스(웹서버, DB 등)
- **Package(패키지)**: 설치 가능한 소프트웨어 단위
- **Repository(저장소)**: 패키지를 제공하는 곳(패키지 소스)
- **Filesystem(파일시스템)**: 파일/디렉토리 저장 구조와 규칙
- **Permission(권한)**: 읽기/쓰기/실행 및 소유자/그룹 기준 접근 제어

---

## 8) 이 문서 다음에 뭘 공부하면 좋을까?

1. **파일/디렉토리 구조 + 기본 명령어**
   - pwd, ls, cd, cat, less, grep, find, head/tail
2. **권한/소유자**
   - chmod, chown, umask, sudo, users/groups
3. **프로세스/서비스 관리**
   - ps, top, kill, systemd/systemctl 개념
4. **네트워크 점검**
   - ip, ss, curl, ping, traceroute, DNS 확인
5. **로그 보는 법**
   - /var/log, journalctl, 애플리케이션 로그 구조


## 참고
- 리눅스는 커널이고, 배포판은 운영 가능한 완제품 OS다.
- 배포판은 “좋고 나쁨”이 아니라 “목적과 운영 철학”이 다르다.
- 실무는 결국 **프로세스/권한/로그/네트워크** 4축으로 돌아간다.
